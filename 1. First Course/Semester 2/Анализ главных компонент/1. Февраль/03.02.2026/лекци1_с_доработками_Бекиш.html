Зачем нужен PCA в Big Data / AI

  • Сотни и тысячи признаков → «проклятие размерности» и шум.[web:17][web:31]
  • Нужен компактный набор осмысленных направлений вариации данных.
  • PCA уменьшает размерность с минимальной потерей информации (в смысле дисперсии) (Smith, ESL). [web:14][web:17]
  • Используется в визуализации, сжатии данных, шумоподавлении, обнаружении аномалий (OpenCV PCA). [web:13][web:31]

  • Дальнейшее развитие: многомерная статистика, анализ данных, машинное обучение.[web:27][web:31]

Связь с градиентными методами (1)

Первая главная компонента задаётся задачей:

\[ \max_{w \in \mathbb{R}^p} \quad w^\top S w \quad \text{при} \quad \|w\| = 1, \]

где \(S\) — ковариационная матрица (Smith, PCA Tutorial, ESL). [web:14][web:17]

  • Это максимум квадратичной формы на единичной сфере.
  • Стационарные точки: \(\nabla (w^\top S w - \lambda (w^\top w - 1)) = 0\) → \(S w = \lambda w\).[web:14][web:17]
  • Главные компоненты — собственные векторы \(S\) с наибольшими собственными значениями.

Связь с градиентными методами (2)

  • Можно применять градиентный подъём по \(w\) с проекцией на сферу (gradient ascent on the sphere).[web:14]
  • Метод степенных итераций (power method) — частный случай итеративного обновления \(w\) вдоль направления \(S w\).[web:17]
  • Сепаратрисы фазового потока градиентной системы разделяют притягивающие множества для разных собственных векторов.
  • На практике используют SVD/спектральные методы для устойчивости и скорости (ESL, Jolliffe & Cadima). [web:17][web:31]

PCA в \(\mathbb{R}^2\): простой пример

  • Имеем облако точек, вытянутое вдоль некоторой диагонали.
  • Первая главная компонента — направление максимальной дисперсии (длинная ось эллипса).[web:14][web:16]
  • Вторая компонента — ортогональное направление с максимальной оставшейся дисперсией.[web:14]
  • Проекция на первую компоненту ≈ одномерное представление данных.

Иллюстрации: Smith, PCA Tutorial.[web:14]

Более сложный пример: изображения (CV)

  • Каждое изображение лица → длинный вектор пикселей.[web:13]
  • PCA на матрице таких векторов даёт eigenfaces — базис типичных вариаций лица.[web:13]
  • Обрезав до первых \(k\) компонент, получаем сжатое/очищенное представление.[web:13]
  • Оптимизационная постановка и спектральное разложение как в \(\mathbb{R}^2\) (OpenCV PCA, ESL). [web:13][web:17]

Формальная постановка PCA

  • Матрица данных: \(X \in \mathbb{R}^{n \times p}\), строки — наблюдения, столбцы — признаки.[web:14][web:17]
  • Центрируем данные: вычитаем среднее по каждому признаку.
  • Ковариационная матрица: \(S = \frac{1}{n-1} X^\top X\) (для центрированных данных).[web:14]
  • Собственные пары \((\lambda_k, w_k)\) ковариационной матрицы дают главные компоненты.[web:14][web:17]

Объяснённая дисперсия и выбор \(k\)

  • Собственные значения \(\lambda_1 \ge \lambda_2 \ge \dots \ge \lambda_p\) — дисперсии компонент.[web:14][web:17]
  • Доля объяснённой дисперсии: \[ \text{EVR}_k = \frac{\sum_{i=1}^k \lambda_i}{\sum_{i=1}^p \lambda_i}. \]
  • Scree‑plot: график \(\lambda_i\) или \(\text{EVR}_k\) для выбора числа компонент (ESL). [web:17]
  • Будет подробно разобрано на последующих лекциях и практиках.

Лекция 1: пример кода в Colab (NumPy PCA)


import numpy as np

# X: n × p, центрированные данные
Xc = X - X.mean(axis=0, keepdims=True)
S  = Xc.T @ Xc / (Xc.shape[0] - 1)

eigvals, eigvecs = np.linalg.eigh(S)  # симметричная матрица
idx = np.argsort(eigvals)[::-1]
eigvals = eigvals[idx]
eigvecs = eigvecs[:, idx]

# Проекции на первые k компонент
k = 2
Z = Xc @ eigvecs[:, :k]

Сравнение с sklearn.decomposition.PCA будет в практике 2 (пример настройки). [web:21]

Лекция 2: линейная алгебра и SVD

  • Ортогональные преобразования и ортонормированные базисы.[web:16][web:17]
  • Собственные значения/векторы симметрических матриц.
  • Сингулярное разложение (SVD — Singular Value Decomposition) \(X = U \Sigma V^\top\).[web:17]
  • Связь между SVD и PCA (столбцы \(V\) → собственные векторы \(S\)) (Smith, ESL). [web:14][web:17]

PCA через SVD

  • Для центрированного \(X\): SVD \(X = U \Sigma V^\top\).[web:17]
  • Ковариационная матрица \(S = \frac{1}{n-1} X^\top X = V \frac{\Sigma^2}{n-1} V^\top\).[web:14]
  • Главные направления (loadings) — столбцы \(V\).
  • Сингулярные значения \(\sigma_i\) связаны с дисперсией компонент: \(\lambda_i = \sigma_i^2/(n-1)\).[web:14][web:17]

Лекция 2: многомерная статистика

  • Многомерное нормальное распределение \( \mathcal{N}(\mu, S)\).[web:17]
  • Эллипсоиды рассеяния и их главные оси.
  • Инвариантность PCA относительно вращений координат.
  • Роль масштабирования (covariance vs correlation matrix) (Smith, обсуждение у Jolliffe & Cadima).[web:14][web:28][web:31]

Пример: влияние масштабов признаков

  • Если один признак измеряется в больших единицах → доминирует в ковариационной матрице.[web:28]
  • Решение: стандартизация (z‑score) или использование корреляционной матрицы.
  • Покажем на практике: без нормировки PCA «видит» только один признак.
  • Детально обсуждается в обзоре Jolliffe & Cadima (2016) .[web:31]

Лекция 2: пример кода (PCA через SVD)


import numpy as np

Xc = X - X.mean(axis=0, keepdims=True)
U, S, Vt = np.linalg.svd(Xc, full_matrices=False)

# V: loadings (principal axes in feature space)
V = Vt.T
eigvals = (S ** 2) / (Xc.shape[0] - 1)

# Проекции (scores)
Z = Xc @ V[:, :2]

Эта формула используется в большинстве реализаций PCA в научных пакетах (ESL). [web:17]

Термины и сокращения (1)

  • PCA — Principal Component Analysis (анализ главных компонент).
  • PC — Principal Component (главная компонента).
  • Explained variance — Объяснённая дисперсия.
  • Loading — Вес признака в главной компоненте (координата признака в векторе компоненты).
  • Score — Координата наблюдения в пространстве главных компонент.
  • Scree plot — График собственных значений или доли объяснённой дисперсии (ESL). [web:17]

Термины и сокращения (2)

  • SVD — Singular Value Decomposition (сингулярное разложение матрицы).[web:17]
  • Covariance matrix — Ковариационная матрица.
  • Correlation matrix — Корреляционная матрица.
  • High-dimensional data — Высокомерные данные (часто \(p \gg n\)).[web:17]
  • Dimensionality reduction — Снижение размерности.
  • Kernel PCA — Kernel Principal Component Analysis (ядерный анализ главных компонент).[web:17]

Термины и сокращения (3)

  • Probabilistic PCA (PPCA) — Probabilistic Principal Component Analysis (вероятностный анализ главных компонент).[web:17]
  • Sparse PCA — Sparse Principal Component Analysis (разреженный анализ главных компонент).[web:31]
  • Q1 journal — Журнал первого квартиля по рейтингу (impact‑фактор, квартиль).
  • Eigenvalue / eigenvector — Собственное значение / собственный вектор.
  • Gradient descent — Градиентный спуск; gradient flow — градиентный поток.
  • Separatrix — Separatrix (сепаратриса, траектория, разделяющая области фазового пространства).

1. Реализовать PCA «вручную»


import numpy as np


x = np.arange(-5, 5)
y = 2 * x + np.random.randn(10) * 3
X = np.column_stack((x, y))

m, n = X.shape

middle_X = []
for col in range(n):
    middle_X += [X[:, col].sum() / m]

middle_X = np.array(middle_X)
X_ = X - middle_X


cov = (X_.T @ X_) / m
N, V = np.linalg.eigh(cov)


Z = X_ @ V[:, N.argsort()[::-1]]

1. Реализовать PCA через scikit‑learn


from sklearn.decomposition import PCA


pca = PCA(n_components=n)
Z = pca.fit_transform(X)
