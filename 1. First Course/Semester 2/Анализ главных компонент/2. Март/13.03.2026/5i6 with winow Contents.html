🧪 Практики 5 и 6: PCA + K‑means, Whitening, t‑SNE

Курс: Анализ главных компонент (СамТУ, М.В. Долгополов)

Материал построен по принципу от простого к сложному. Вы повторите ключевые понятия, разберёте примеры и выполните самостоятельные задания с ответами.

📌 Что необходимо знать перед началом?

  • PCA — линейное преобразование для уменьшения размерности. Главные компоненты — собственные векторы ковариационной матрицы. Доля объяснённой дисперсии (EVR) = \( \lambda_i / \sum \lambda_j \).
    Формула проекции: \( Z = X_{\text{std}} W \), где \( W \) — матрица собственных векторов.
  • Стандартизация — обязательный шаг перед PCA (признаки приводятся к нулевому среднему и единичной дисперсии).
  • K‑means — алгоритм кластеризации, минимизирующий инерцию (сумму квадратов расстояний до центроидов).
  • Метод локтя (Elbow method) — выбор k по изгибу графика инерции.
  • Силуэт (Silhouette) — метрика качества кластеризации (от -1 до 1).
  • Biplot — совмещённое отображение объектов (точки) и нагрузок исходных признаков (стрелки).

🔬 Практика 5: PCA как предобработка для кластеризации

Датасет: Mall Customers (Age, Annual Income (k$), Spending Score (1-100)).
Цель: Сравнить кластеризацию K‑means на исходных признаках и на главных компонентах. Оценить влияние PCA.

5.1 EDA и стандартизация

Загружаем данные (например, из Mall_Customers.csv). Первичный анализ: распределения, парные графики. Все признаки имеют разные масштабы, поэтому применяем StandardScaler.

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

df = pd.read_csv('Mall_Customers.csv')  # предположим, столбцы: Age, Income, Spending
X = df[['Age', 'Annual Income (k$)', 'Spending Score (1-100)']]
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

5.2 PCA: объяснённая дисперсия и выбор числа компонент

Обучаем PCA на стандартизированных данных, смотрим долю объяснённой дисперсии каждой компоненты (EVR) и накопленную дисперсию.

pca = PCA()
X_pca = pca.fit_transform(X_scaled)
evr = pca.explained_variance_ratio_
cumsum = np.cumsum(evr)
print('EVR:', evr)
print('Накопленная дисперсия:', cumsum)

plt.figure(figsize=(8,4))
plt.subplot(1,2,1)
plt.bar(range(1,4), evr, tick_label=['PC1','PC2','PC3'])
plt.ylabel('Доля дисперсии')
plt.subplot(1,2,2)
plt.plot(range(1,4), cumsum, 'o-')
plt.xlabel('Число компонент')
plt.ylabel('Накопленная дисперсия')
plt.show()
📊 Рисунок 1: Графики EVR и накопленной дисперсии. Обычно первые две компоненты объясняют ~60-70% дисперсии.

Вывод: для визуализации возьмём 2 компоненты. Для кластеризации можно оставить 2 или 3 (зависит от задачи).

5.3 K‑means на исходных признаках

Перебираем k от 2 до 10, вычисляем инерцию и силуэт.

inertia = []
silhouette = []
K_range = range(2, 11)
for k in K_range:
    km = KMeans(n_clusters=k, random_state=42)
    labels = km.fit_predict(X_scaled)
    inertia.append(km.inertia_)
    silhouette.append(silhouette_score(X_scaled, labels))

plt.figure(figsize=(12,4))
plt.subplot(1,2,1)
plt.plot(K_range, inertia, 'bo-')
plt.xlabel('k'); plt.ylabel('Инерция'); plt.title('Метод локтя')
plt.subplot(1,2,2)
plt.plot(K_range, silhouette, 'ro-')
plt.xlabel('k'); plt.ylabel('Силуэт'); plt.title('Silhouette score')
plt.show()
📊 Рисунок 2: Локоть наблюдается при k=5, силуэт максимален при k=5 (около 0.45). Выбираем k=5.

Получаем кластеры на исходных данных. Запомним метки labels_orig.

5.4 K‑means на главных компонентах

Повторим кластеризацию, но теперь на первых двух главных компонентах (можно и на трёх). Сравним метрики.

X_pca2 = X_pca[:, :2]  # две компоненты
inertia_pca = []
silhouette_pca = []
for k in K_range:
    km = KMeans(n_clusters=k, random_state=42)
    labels = km.fit_predict(X_pca2)
    inertia_pca.append(km.inertia_)
    silhouette_pca.append(silhouette_score(X_pca2, labels))

# Сравнение силуэта
plt.plot(K_range, silhouette, label='Исходные признаки')
plt.plot(K_range, silhouette_pca, label='2 главные компоненты')
plt.xlabel('k'); plt.ylabel('Silhouette'); plt.legend(); plt.show()
📊 Рисунок 3: Сравнение силуэта. Обычно значения близки, но на PC может быть немного ниже из‑за потери информации. Однако шум может уменьшиться, и кластеры станут более интерпретируемыми.

Вывод: PCA может служить хорошей предобработкой для кластеризации, особенно если признаки коррелированы. Это уменьшает размерность и шум.

5.5 Biplot кластеров

Строим scatter первых двух главных компонент, окрашивая точки по кластерам (например, полученным на исходных данных). Добавляем стрелки — нагрузки исходных признаков.

# Получаем нагрузки (компоненты)
loadings = pca.components_.T * np.sqrt(pca.explained_variance_)  # для масштаба как в биплоте
# или просто pca.components_.T

plt.figure(figsize=(10,8))
scatter = plt.scatter(X_pca[:,0], X_pca[:,1], c=labels_orig, cmap='viridis', alpha=0.6)
for i, feature in enumerate(['Age','Income','Spending']):
    plt.arrow(0, 0, loadings[i,0], loadings[i,1], color='red', alpha=0.8, head_width=0.1)
    plt.text(loadings[i,0]*1.2, loadings[i,1]*1.2, feature, color='red', fontsize=12)
plt.xlabel('PC1')
plt.ylabel('PC2')
plt.colorbar(scatter, label='Кластер')
plt.title('Biplot кластеров (K‑means на исходных данных)')
plt.grid(True, alpha=0.3)
plt.show()
📊 Рисунок 4: Biplot показывает, что PC1 в основном определяется доходом и тратами (противоположно возрасту?), а PC2 — возрастом. Кластеры хорошо разделяются.

⚡ Практика 6: Whitening и t‑SNE

6.1 PCA Whitening (отбеливание)

Определение: Преобразование, после которого ковариационная матрица данных становится единичной (\( \text{cov}(Z_{\text{whitened}}) = I \)). Это делает данные "сферическими" — все направления равнозначны. Полезно для алгоритмов, чувствительных к масштабу (нейронные сети, SVM).

\[ Z_{\text{whitened}} = Z \cdot \Lambda^{-1/2} \]

где \( Z \) — матрица проекций на главные компоненты (после PCA), а \( \Lambda = \text{diag}(\lambda_1, \dots, \lambda_k) \) — собственные значения (дисперсии вдоль компонент). Произведение делается поэлементно: каждый столбец \( Z \) делится на корень из соответствующего собственного значения.

В коде (после PCA):

# X_pca — проекции на все компоненты (k=3)
lambdas = pca.explained_variance_  # собственные значения
X_whitened = X_pca / np.sqrt(lambdas)  # broadcasting по столбцам
# Ковариация X_whitened должна быть близка к единичной
print(np.cov(X_whitened, rowvar=False).round(3))

🔗 Whitening transformation (Wiki)

Примечание: Существует также ZCA‑whitening, который возвращает данные в исходное пространство признаков, но с единичной ковариацией. PCA whitening (просто масштабирование компонент) — самый простой вариант.

6.2 t‑SNE vs PCA

PCAt‑SNE
Линейное преобразованиеНелинейное (основано на вероятностях)
Быстрое (O(n·d²))Медленное (O(n²))
Сохраняет глобальную структуру (максимум дисперсии)Сохраняет локальную структуру (близость соседей)
Детерминированный результатСтохастический (разный при разных запусках)
Можно использовать как этап модели (признаки)Только для визуализации (не для построения моделей)

Основная идея t‑SNE: преобразует расстояния в вероятности и минимизирует расхождение Кульбака‑Лейблера между распределениями в исходном и низкоразмерном пространствах.

6.3 Визуализация кластеров на Mall Customers

Применим t‑SNE к стандартизированным данным и сравним с PCA (2 компоненты). Раскрасим точки по кластерам, полученным в практикуме 5 (например, k=5).

from sklearn.manifold import TSNE

tsne = TSNE(n_components=2, random_state=42, perplexity=30)
X_tsne = tsne.fit_transform(X_scaled)

plt.figure(figsize=(12,5))
plt.subplot(1,2,1)
plt.scatter(X_pca[:,0], X_pca[:,1], c=labels_orig, cmap='tab10', alpha=0.6)
plt.title('PCA (2 компоненты)')
plt.subplot(1,2,2)
plt.scatter(X_tsne[:,0], X_tsne[:,1], c=labels_orig, cmap='tab10', alpha=0.6)
plt.title('t‑SNE (perplexity=30)')
plt.show()
📊 Рисунок 5: Сравнение PCA и t‑SNE. PCA даёт глобальное разделение по осям (доход/возраст), t‑SNE часто показывает более чёткие локальные кластеры, иногда даже подкластеры.

Выводы: t‑SNE лучше подходит для визуализации сложных нелинейных структур, но результат зависит от настройки параметров (perplexity). PCA даёт быструю и интерпретируемую линейную проекцию.

⚠️ t‑SNE не следует использовать для уменьшения размерности перед кластеризацией — он оптимизирован для визуализации и искажает глобальные расстояния.

Задание 1: Whitening + K‑means

На датасете Mall Customers примените PCA‑whitening (к первым двум компонентам) и выполните K‑means (k=5). Сравните силуэт с кластеризацией на обычных PC. Улучшилось ли качество? Почему?

🔍 Подсказка После whitening все компоненты имеют одинаковую дисперсию (1). K‑means опирается на евклидовы расстояния, поэтому равный масштаб может быть полезен. Попробуйте и сравните.
✅ Ответ (пример) Код: 
X_pca2 = X_pca[:, :2]
lambdas2 = pca.explained_variance_[:2]
X_white2 = X_pca2 / np.sqrt(lambdas2)
km_white = KMeans(n_clusters=5, random_state=42).fit(X_white2)
sil_white = silhouette_score(X_white2, km_white.labels_)
print('Силуэт на whitened PC:', sil_white)
# Обычно значение близко к кластеризации на PC, но может немного измениться из‑за перемасштабирования.

Whitening не всегда улучшает кластеризацию, но может помочь, если данные сильно анизотропны. В нашем случае разница незначительна.

Задание 3: Эксперименты с perplexity в t‑SNE

На Mall Customers постройте t‑SNE с разными значениями perplexity (5, 30, 50, 100). Как меняется визуальная картина? Сравните с PCA.

🔍 Подсказка Perplexity можно интерпретировать как эффективное число соседей. Слишком малое значение приводит к "шумному" разбросу, слишком большое — к вырождению в однородное облако.
✅ Ответ (пример) 
for perp in [5,30,50,100]:
    tsne = TSNE(n_components=2, perplexity=perp, random_state=42)
    Xt = tsne.fit_transform(X_scaled)
    plt.scatter(Xt[:,0], Xt[:,1], c=labels_orig, cmap='tab10', alpha=0.6)
    plt.title(f'perplexity={perp}')
    plt.show()

При малом perplexity кластеры дробятся, при большом — сливаются. Perplexity=30 даёт обычно разумный компромисс.
