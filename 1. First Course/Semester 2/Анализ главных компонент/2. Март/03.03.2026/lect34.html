Словарь сокращений и терминов

  • PCA — Principal Component Analysis, анализ главных компонент.
  • PC — Principal Component, главная компонента, вектор в пространстве признаков.
  • Scores — координаты объектов в пространстве главных компонент, строки матрицы \(Z\).
  • Loadings — веса признаков в компонентах, столбцы матрицы \(W\) (собственные векторы ковариационной матрицы).
  • EV — Eigenvalue, собственное значение.
  • EVR — Explained Variance Ratio, доля объяснённой дисперсии.
  • Scree‑plot — график собственных значений или EVR по номеру компоненты.
  • Biplot — график, где одновременно показаны scores (объекты) и loadings (признаки) в пространстве компонент.
  • Whitening — «выбеливание»; линейное преобразование, делающее ковариационную матрицу близкой к единичной.

Explained variance: формальные определения

  • Пусть ковариационная матрица: \[ S = \frac{1}{n-1}\tilde{X}^\top \tilde{X}, \] её собственные значения \(\lambda_1 \ge \dots \ge \lambda_p\).
  • Общая дисперсия: \[ \mathrm{TotalVar} = \sum_{i=1}^p \lambda_i. \]
  • Доля дисперсии, объяснённая компонентой \(k\): \[ \mathrm{EVR}_k = \frac{\lambda_k}{\sum_{i=1}^p \lambda_i}. \]
  • Кумулятивная доля дисперсии: \[ \mathrm{CumEVR}_k = \sum_{i=1}^k \mathrm{EVR}_i. \]

Explained variance: интуиция

  • Каждая PC — это направление, вдоль которого точки «разбросаны» максимально широко.
  • \(\lambda_k\) — дисперсия данных при проекции на направление \(w_k\).
  • EVR говорит, какую долю общей изменчивости данных мы сохраняем, если смотрим только на компоненту \(k\).
  • Чем больше суммарный EVR первых \(k\) компонент, тем меньше информации теряется при переходе в \(k\)-мерное подпространство.

Интерактив: калькулятор EVR

Введите собственные значения (через пробел или запятую), например: 4.0 1.0 0.5 0.2.

Пример: если \(\lambda = (4,1,0.5,0.2)\), то EVR\(_1\) ≈ 68%, EVR\(_2\) ≈ 17%, EVR\(_3\) ≈ 9%, EVR\(_4\) ≈ 3%.

Scree‑plot: идея

  • Scree‑plot — график \(\lambda_i\) или \(\mathrm{EVR}_i\) по номеру компоненты \(i\).
  • Смотрим на «колено» кривой: после него добавление компонент даёт малый прирост объяснённой дисперсии.
  • Для Iris scree‑plot показывает, что первые 2 компоненты уже объясняют большую часть вариации.
  • Рекомендуемый разбор scree‑plot: How to Make a Scree Plot with Python.

Iris: scree‑plot (код на Python)


from sklearn.decomposition import PCA
from sklearn.datasets import load_iris
import matplotlib.pyplot as plt

iris = load_iris()
X = iris["data"]

pca = PCA().fit(X)
eigvals = pca.explained_variance_
evr = pca.explained_variance_ratio_
cum_evr = evr.cumsum()

fig, ax = plt.subplots(1, 2, figsize=(10, 4))

ax[0].plot(range(1, len(eigvals)+1), eigvals, marker="o")
ax[0].set_xlabel("Компонента")
ax[0].set_ylabel("λ (собственное значение)")
ax[0].set_title("Scree-plot (Iris)")

ax[1].plot(range(1, len(cum_evr)+1), cum_evr, marker="o")
ax[1].set_xlabel("k")
ax[1].set_ylabel("CumEVR")
ax[1].set_title("Cumulative EVR (Iris)")

plt.tight_layout()
plt.show()

Выбор числа компонент: критерии

  • Критерий доли дисперсии. Выбираем минимальное \(k\), для которого \(\mathrm{CumEVR}_k \ge \tau\), где \(\tau\) — порог (например, 0.9 или 0.95).
  • Критерий «колена» scree‑plot. Ищем визуальный излом на графике \(\lambda_i\), после которого значения убывают медленно.
  • Критерий Kaiser (для стандартизованных данных). Берём компоненты, у которых \(\lambda_i > 1\) (для данных с единичной дисперсией признаков).
  • В реальных задачах критерии комбинируют с доменной интерпретацией и требованиями к визуализации.

Wine: пример выбора числа компонент


from sklearn.datasets import load_wine
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

wine = load_wine()
Xw = wine["data"]

pca_w = PCA().fit(Xw)
eigvals_w = pca_w.explained_variance_
evr_w = pca_w.explained_variance_ratio_
cum_evr_w = evr_w.cumsum()

print("EVR:", evr_w)
print("CumEVR:", cum_evr_w)
k90 = (cum_evr_w >= 0.9).argmax() + 1
print("k для CumEVR ≥ 0.9:", k90)

Сравните, сколько компонент нужно для 90% дисперсии на Iris и на Wine: на Wine обычно требуется больше компонент, т.к. исходная размерность выше и структура сложнее.

Biplot: определение

  • Пусть после PCA имеем: \[ Z = \tilde{X} W, \quad Z \in \mathbb{R}^{n \times p}, \quad W \in \mathbb{R}^{p \times p}. \] Рассмотрим первые две компоненты: \(Z_{(2)} \in \mathbb{R}^{n \times 2}\), \(W_{(2)} \in \mathbb{R}^{p \times 2}\).
  • Scores — точки \((z_{i1}, z_{i2})\), \(i=1,\dots,n\): объекты в координатах PC1–PC2.
  • Loadings — векторы \((w_{j1}, w_{j2})\), \(j=1,\dots,p\): признаки в том же пространстве.
  • Biplot — совместный график scores и loadings в плоскости первых компонент.

Biplot: как его читать

  • Стрелка признака показывает направление увеличения значения признака.
  • Длина стрелки пропорциональна тому, насколько признак хорошо «виден» в пространстве первых компонент (по модулю корреляции с компонентами).
  • Угол между стрелками приблизительно соответствует корреляции: маленький угол — сильная положительная корреляция, угол ~90° — слабая связь, ~180° — отрицательная корреляция.
  • Положение кластеров объектов относительно стрелок говорит, какие признаки их отличают.

Iris: biplot (код)


import numpy as np
import matplotlib.pyplot as plt
from sklearn.decomposition import PCA
from sklearn.datasets import load_iris

iris = load_iris()
X = iris["data"]
y = iris["target"]
features = iris["feature_names"]

Xc = X - X.mean(axis=0, keepdims=True)
pca = PCA(n_components=2)
Z = pca.fit_transform(Xc)      # scores (n×2)
W = pca.components_.T          # loadings (p×2)

plt.figure(figsize=(7, 7))
plt.scatter(Z[:, 0], Z[:, 1], c=y, cmap="viridis", alpha=0.7)

for i, f in enumerate(features):
    plt.arrow(0, 0, W[i, 0]*3, W[i, 1]*3, color="r", head_width=0.05)
    plt.text(W[i, 0]*3.2, W[i, 1]*3.2, f, color="r")

plt.xlabel("PC1")
plt.ylabel("PC2")
plt.axhline(0, color="grey", linewidth=0.5)
plt.axvline(0, color="grey", linewidth=0.5)
plt.title("PCA Biplot (Iris)")
plt.grid(True, alpha=0.2)
plt.show()

Iris: интерпретация biplot

  • Стрелки лепестков (petal length/width) обычно тянут вдоль PC1: PC1 ≈ «размер лепестков».
  • Стрелки чашелистиков (sepal) больше влияют на PC2: PC2 ≈ «пропорции чашелистик/лепесток».
  • Setosa часто отделена от остальных по PC1 и/или PC2, Versicolor и Virginica частично перекрываются.
  • Это уже даёт биологический смысл компонентам: не просто «абстрактная ось», а конкретный морфологический фактор.

Wine: идея biplot

  • Датасет Wine: химические параметры вин (алкоголь, кислоты, минералы и т.п.).
  • PCA позволяет выделить компоненты, связанные, например, с «общей насыщенностью», «кислотностью», «минеральностью».
  • Biplot в PC1–PC2 помогает увидеть, какие признаки отличают сорта (classes 0, 1, 2) и насколько они разделимы.
  • Отличная площадка для самостоятельного исследования и попытки интерпретировать каждую компоненту в химических терминах.

Типичные ошибки при интерпретации PCA/biplot

  • Считать, что PC1 всегда имеет физически осмысленную интерпретацию — иногда это чисто математический компромисс разных эффектов.
  • Игнорировать масштабирование признаков: без стандартизации признаки с большой дисперсией доминируют.
  • Переинтерпретировать biplot: заключать о причинно‑следственных связях, видя только корреляции и геометрию.
  • Забывать о том, что biplot в PC1–PC2 не показывает вклад признаков в последующие компоненты.

Итог лекций 3 и 4

  • Explained variance / EVR — количественная мера того, сколько вариации захватывает каждая компонента.
  • Scree‑plot и cumulative EVR — ключевые инструменты выбора числа компонент.
  • Biplot — мощный визуальный метод совместной интерпретации объектов и признаков.
  • Интерактивные калькуляторы и небольшие численные примеры помогают «почувствовать» PCA, а не только выучить формулы.
